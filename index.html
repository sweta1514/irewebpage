Problem Statement

The aim of the project is to create a model for generating summaries of text documents with respect to a given query. This summary generation is know as Query-based Summarization.

Introduction

  • Types of Summarization:

    • Extractive Summarization:

      • It selects important sentences from the text to form a summary. Thus it produces a summary which contains parts of the original document.

      • In such a summarization there are inconsistencies in meaning and context & the sentences aren’t essentially connected.

    • Abstractive Summarization:

      • Abstractive summarization uses different models and NLP techniques to form a coherent summary.

      • Such a summary is comparable to a human-written summary which makes it easier for the reader to understand and doesn’t lose out on the context and the meaning.

  • A significant difference in the model we present is that it generates an abstractive summary. It allows for re-phrasing and using words not necessarily present in the original document.

  • In Query based abstractive text summarization the aim is to generate the summary of a document in the context of a query. In general, abstractive summarization, aims to cover all the salient points of a document in a compact and coherent fashion.

  • Query focused summarization highlights those points that are relevant in the context of the query.

Applications

  • In search engines :-

    • When we search for a query in our favourite browser it shows a list of links and brief summaries related to the query.

  • Increases the ability to answer a query which is posted in Natural Language.

  • Brief understanding of a text/document.

Results & Dataset

  • Summaries are considered complete when a specific token is generated.

  • The dataset used in this purpose consists of document–query–answer triples from CNN and Daily Mail news articles.
